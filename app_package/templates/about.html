About Dashboards and Databases


Mission

Create a process that feels as though you have a team with one more analyst that has a programming skill set so we can build the application/tool together.


What is a dashboard and database (DD) application?

A simple way to think of a DD application is a well-organized excel spreadsheet. When the spreadsheet is first created it’s a clean table with highlighted column headings nicely fitted rows and easy to read text. But as the spreadsheet gets passed around your team columns/rows get added, new highlights are made, text no longer fits in each cell readable and comments in emails included in the circulation email are critical making the spreadsheet no longer a standalone document.

A DD application takes that messy spreadsheet puts everything into a database with a clean display. Then a user interface is built around the database allows users to view data items, make changes and add new information. Then view the data again in the application or build reports in excel, text, or some other media.

All this can be done with authentication and/or different user privileges so its accessible to only those desired and protected from those outside the team.

Some past work

Here are three demo applications:
  • Kinetic Metrics/ NHTSA Dashboard
    • Dashboard built around a dataset provided by the National Highway Traffic Safety Administration and available at https://www-odi.nhtsa.dot.gov/downloads/.
  • Shiftgarten
    • Dashboard built around restaurant management records. All data is user input. The structure is built around how many restaurants and what type of data the user wants to collect. Data is from restaurant revenue and costs to employee hours and pay.
  • What Sticks Health
    • Dashboard built around personal device data. The goal of this platform is to allow for users to have centralized data storage from everything from their Fitbit steps to their 23andMe gene sequence to recording how many cups of coffee they drink a day. With all the data stored in one place the user will be able to gain perspective on what sorts of things impact their health. Currently the demo shows heart rate readings from Polar device and inputs on coffee intake. This product is still under construction.
All these applications have similar concepts of a user interface with a backend database. However, each application has unique functionality. Hopefully, this provides some intuition for how your project can benefit from a DD application.
